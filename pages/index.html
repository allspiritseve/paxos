# Message Id The paxos message id for a cluster will increment indefinately. It is a hexidecimal string representing an integer. So long as it is less than 252, you can parse it into a JavaScript number for comparison. You can store it as a `BITINT` in MySQL so long as it is less than 264. You can use your own key, of course, but you also, always order using the paxos `Identifier` class. TODO: https://gist.github.com/mayoff/1138816
